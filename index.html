MRT Breadth First Search

Context: Singapore MRT system.

Description: Our Singapore Mrt System has grown rather complicated over the past few years. It is also well known that transport systems can be modelled by graphs. Hence some people might be interested in the 'connectivity" of our mrt system, that is, the number of possible distinct acyclic paths are there between 2 stations (only limited to the 4 main lines, ignore the LRTs)? (may be interesting for people planning their trips to school from home, can also be useful for alternative path if one day the mrt breaks down). link to mrt route map (pdf)
Possible extension is to print the path out.

Learning Objectives: Simple graph theory (modeling) and BFS

Relevant CS topics/focus: Enumerating paths on a graph.

Algorithms involved (if any): BFS. We will have to teach them BFS and probably guide them on modeling this problem effectively. The idea is to focus on the Interchange stations on the 4 main lines.

Scenario 2

Context: (idea shamelessly taken from here.) There's a paper by Thomas Schelling called "Dynamic Models of Segregation", which proposes a simple model of racial segregation. Let's code our own model of it.

Description: The Schelling model of the city is an array of cells where each cell represents a house. The houses are occupied by two kinds of agents, labeled red and blue, in roughly equal numbers. About 10% of the houses are empty.

At any point in time, an agent might be happy or unhappy, depending on the other agents in the neighborhood. In one version of the model, agents are happy if they have at least two neighbors like themselves, and unhappy if they have one or zero.

The simulation proceeds by choosing an agent at random and checking to see whether it is happy. If so, nothing happens; if not, the agent chooses one of the unoccupied cells at random and moves.

Learning Objectives: Learn how to use computational models to model real world behavior.

Relevant CS topics/focus: Agent-based modeling.

Context: Cheapest and fastest flight.

Description: For this scenario, a list of starting, ending location and price will be given. For example (Singapore, Bali, 200), which means the flight from Singapore to Malaysia cost $200. Soedarsono

Movie Marathon

Context: In the United States, movie theatres work slightly differently. Your movie ticket doesn't contain a seat number, and this means that there are free seating in the movie theatre. In addition, after a movie has ended, there is no one to guide you to the exit. This means that after your movie has ended, you can *technically* slip into another movie theatre to watch another movie.

Description: Given a list of timings for different movies, determine the best way to watch as many movies as possible with the least possible waiting time in between movies.

Learning Objectives:

  • Iteration
  • Handling edge cases

Relevant CS topics/focus: Looping

Route Planning

Context: Planning public transportation routes to go from point A to point B might involve multiple transfers, and it can be tedious to determine the "best" possible way to get from point A to point B in time. Given all the data, a computer can enumerate through all the possible routes that one can take, and show the "best" routes.

Description: Given:

  1. a list of locations
  2. all train/bus routes
  3. arrival time of the train/bus at each location
  4. starting location and ending location
  5. latest time to reach end location

We want to be able to determine the best possible route to arrive at the ending location.

The best possible routes can be defined by:

  • Cost of the route
  • Least number of transfers
  • Latest possible time to leave starting location

Learning Objectives:

  • OOP: organising the data in the nodes
  • Graph as a way to solve problem

Relevant CS topics/focus: Graphs and Optimization
