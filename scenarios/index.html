Agent-based Modeling

Context: (idea shamelessly taken from here.) There's a paper by Thomas Schelling called "Dynamic Models of Segregation", which proposes a simple model of racial segregation. Let's code our own model of it.

Description: The Schelling model of the city is an array of cells where each cell represents a house. The houses are occupied by two kinds of agents, labeled red and blue, in roughly equal numbers. About 10% of the houses are empty.

At any point in time, an agent might be happy or unhappy, depending on the other agents in the neighborhood. In one version of the model, agents are happy if they have at least two neighbors like themselves, and unhappy if they have one or zero.

The simulation proceeds by choosing an agent at random and checking to see whether it is happy. If so, nothing happens; if not, the agent chooses one of the unoccupied cells at random and moves.

Learning Objectives: Learn how to use computational models to model real world behavior.

Relevant CS topics/focus: Agent-based modeling.

Airport Gates

Context: Gate assignment problem in an airport.

Description: After an aircraft has landed until the time when it departs again, the airport needs to take care of it by assigning a gate to it. Some work needs to be done during this time. For example, refueling the aircraft, unloading passengers, cleaning, providing new supplies and boarding new passengers. Given the schedules of all aircrafts and a set of gates, how can an airport robustly pair them without failing to meet a set of criteria?

The criteria may include:

  • One gate can only serve one aircraft at a time.
  • Two adjacent gates cannot serve large aircraft at the same time.
  • Some gates can only server certain types of aircraft.
  • Two adjacent gates cannot server aircrafts with same departure time.

Also, a flight (aircraft) may have various properties:

  • Departure time
  • Departure airport
  • Arriving time
  • Destination airport
  • Type
  • Size

Same for a gate:

  • Types of flights it can handle
  • Type
  • Number, location etc.

To achieve robustness, the goal is to maximize the idle time between all pairs of consecutive flights at a gate, ensuring that each flight can arrive either a bit too early or a bit too late without the need for re-planning the schedule. For example, a better solution to the following schedule is to move Flight 3 to Gate 2 and Flight 4 to Gate 1.

A detailed problem formulation and solution used in Amsterdam Airport Schiphol (AAS) can be found here.

Learning Objectives: Optimizing within constraints

Taxi Dispatch

Context: Transportation dispatching systems.

Description: A similar scenario is taxi ordering and dispatching system. Based on the current taxi dispatch system in Singapore, there’re two categories of booking: current and advanced. Current bookings are requests for a taxi to reach a customer immediately or within 30mins. Currently it’s handled based on nearest-location method. A geographically nearest taxi is assigned to pick up the customer. Advanced bookings must be made over half an hour ago. Then the message is spread through out all taxis, both empty and occupied. The job is assigned to the taxi driver who first bid for it.

However, the way of handling current bookings is not always efficient. The taxis must take into account the current traffic condition. Therefore, the one with the shortest-time path should be selected. (Involves SSSP algorithm which may be a hard for freshmen.)

To optimize advanced bookings handling, a taxi with a sequence of advanced booking pickup tasks should minimize the total travel cost, without violating precedence, capacity and time windows constraints (maximum amount of time to reach a customer). The problem can be modeled into Pickup and Delivery Problem with Time Window (PDPTW). The idea is to chain several bookings with pick-up time points spread over a reasonable period of time and to let each pick-up point coincide with or be closed to the previous drop-off location.

EdgeRank Algorithm

Context: Almost everybody uses Facebook everyday. This question will give students deeper insights on how their Facebook's newsfeed is populated based on their Facebook action history.

Description: Students will be tasked to implement a simplified version of Facebook's EdgeRank algorithm which decides which stories appear in their newsfeed.

The three ingredients of the EdgeRank algorithm:

  1. Affinity Score (Activities and the people involved)
  2. Edge Weight (Type of activities)
  3. Time Decay (The older the activity, the lower the score)

Data for user activities (users involved, type of activity, time of activity) will be provided. Based on all these data, the EdgeRank score for each activity can be computed.

Given data of users and their social activities, determine the top-ranked activities that will be displayed for the particular user's newsfeed.

Learning Objectives: Understand the basics of the core algorithms that power major technology startups like Facebook.

Relevant CS topics/focus: Algorithms, Data structures.

Algorithms involved (if any): Simplified version of EdgeRank Algorithm.
